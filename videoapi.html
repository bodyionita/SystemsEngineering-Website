Research into Video Call API's

As our client needs us to build a demonstration site to show off the feature of the "At Home" system, we need to find a way to achieve the 2-way video consultation functionality. The client has said that we may be able to use some existing code from DotComDoctor, however this may not be available until after the demo. Therefore we need to find alternatives to overcome this problem.

After conducting research into available video API's that we could use we found the following options, we then evaluated each one based on some rough criteria below:

  • Free to use
  • No watermarks or branding
  • No requirement for users to create accounts
  • Ability to take screenshots of video feed
  • Well documented API

VSee

Pros Cons
30-day Free Trial User needs to install VSee into their browser
Well Documented API Complicated installation process for users
Used by many telemedicine companies No simple way to take screenshots

SightCall

Pros Cons
Cross-Platform, works on mobile and web Plugin required in browser
Well documented examples Need a "Standard" account to develop
Users don't need to create an account No apparant way to take screenshots

Skype

Pros Cons
Cross-Platform, works on mobile and web Requires a plugin for video/audio
Well documented examples Skype UI looks too familiar
Works on IE, Chrome, Firefox, Safari No apparant way to take screenshots

TokBox

Pros Cons
Well documented API with tutorials and examples $50/month base fee
Well documented examples Skype UI looks too familiar
- No apparant way to take screenshots

ooVoo

Pros Cons
Free version available Free version serves ads
Paid version is by usage only ($0.005 per minute) Documentation is sparse although does exist
Provides a way to take screenshots -
No branding or logos on the video call -
Users don't need to create an account -

Conclusion

After looking at some of our available options it seems like ooVoo provides all the features we will need. However it may be an issue as the only free version of ooVoo will display ads which will not be great for the demonstration. Therefore we may have to use the paid version. The other alternatives either are paid or have similar downsides.

At this point we will continue our research into video API's, we will be experimenting with the ooVoo API and finding further alternatives in time for the demonstration in late-January.
