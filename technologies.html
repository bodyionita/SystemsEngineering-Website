Technologies Research

In order to develop our proof-of-concept websites we needed to pick adequate technologies to develop the system. This included which frameworks/technologies to use for the backend, database and frontend. We drew up a list of pros and cons for each of the items we researched:

Backend Options

Flask

Pros Cons
Previous experience Minimal features
Uses Python Not suited to larger applications
Lots of simple tutorials Not scalable very well

Node.js

Pros Cons
Asynchronous event driven I/O No prior experience
Lots of available learning resources Lots of unnecessary features
Many packages available via npm
Suited for larger applications

Ruby on Rails

Pros Cons
Lots of tools available No prior experience
Moderate amount of learning resources Not familiar with the language
Many packages available via npm Supposedly slow relative runtime speeds

Database Options

SQLite3

Pros Cons
Simple, lightweight Not scalable
Fast SQL engine
Documentation on usage with Flask

SQLAlchemy

Pros Cons
Simple, lightweight and fast More complex when doing simple operations in Flask
Documentation on usage with Flask

Conclusion

After looking at some of the options available to us, we decided to go with Flask for our backend as we had used it extensively in the past and we knew how to get started with it straight away. This would reduce the amount of time needed to learn the technology, giving us more time to focus on developing features. We did not research any other front-end options, we decided that Bootstrap would be the simplest option and we already knew it could provide all the features we needed. Regarding the database, we chose to go with SQLite3 as we had little experience with databases thus far and there were plenty of resources available for SQLite3. Furthermore we only required a simple database with a few tables and so we did not need many of the additional features other technologies were offering.
