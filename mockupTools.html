Mock-up Tools Research

In order to create the UI mock-ups that we required for the client we needed a tool. We had only heard of and used one tool briefly before, Balsamiq. However we did some research to find out if there were better alternatives that suited our needs better.

Our main requirements for the tool included:

  • Able to create dynamic pages
  • Able to easily show to the client
  • Fast and simple interface to create mock-ups
  • Free of charge

We researched the following tools and evaluated each one:

UXPin

Pros Cons
Free Trial Limited free trial then $19 a month
Interactive or Static mock-ups Never used before
Simple Clean UI -
Advertised as "rapid" -
Built in web/mobile elements including Bootstrap -
Project Collaboration with notes and tracking -

Balsamiq

Pros Cons
Free Trial Limited free trial
Interactive mock-ups Not as modern looking as other options
Simple Clean UI -
Project Collaboration with notes and tracking -
2 different skin types - Wireframe or Sketch -
We have used this before -

Axure

Pros Cons
Free Trial Limited free trial to 30 days
Dynamic content and animations More complex UI to create mock-ups
Math functions to change labels dynamically -
Project Collaboration with notes and tracking -
Mock-ups are cloud hosted and easy to share -

Fluid

Pros Cons
Free Trial Limited free trial
16+ built in libraries for designs Focussed on mobile apps rather than web
Supports gestures and transitions between screens -
Project Collaboration with notes and tracking -

Pidoco

Pros Cons
Free Trial Limited free trial
Drag and drop interface Doesn't include common libraries like Bootstrap
Supports gestures and transitions between screens No dynamic mock-ups option
- Can only share/export via PDF

Conclusion

After looking at some of the main options out there we narrowed down our choices to either UXPin, Balsamiq or Azure. These 3 options were all very similar and so it came down to which was easiest to get started with. Therefore we settled on Balsamiq as we had used it previously and knew how it worked. It would be most efficient to just go back to Balsamiq and quickly create mock-up drafts.
